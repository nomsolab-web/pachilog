# パチログ 月1万円達成プラン

「攻略・期待値・投機は一切扱わない」「1日1〜2時間」「初期投資ゼロ」の制約の中で、月1万円の収益を狙うための現実的なロードマップ。

## 1. まず目標値を現実の数字で置く

着想元の「ソーシャルXP」の実績(動画内の公開データ):
- 月間ユーザー約4,457人、月間PV約24,231(1人あたり6〜10ページ回遊)
- AdSense収益予想: 月**5,000〜12,549円**

→ **パチログで月1万円を狙うなら、ざっくり「月間PV2万〜3万・月間ユーザー4,000〜5,000人」が最初のマイルストーン。** SEOに頼らずXシェアのバイラルループだけでこの数字を作れた実例があるので、同じ型を踏襲するのが最短ルート。

## 2. 収益源(優先順位順)

### ① Google AdSense(メイン収益源、今の設計の中心)
- 施策は「③トラフィック成長策」がそのまま収益に直結する
- 審査には独自ドメイン推奨(前述の通り)。コンテンツ量とアクセスがある程度育ってから申請

### ② Xでのバイラルループ強化(コストゼロ、最優先で伸ばすべき部分)
- 今すでに実装済みの「Xでシェア」ボタンを核に、**回遊率を上げる導線を追加**する:
  - トップページに「注目チャンネル」「急上昇/急降下」に加えて「新着投票」「今週伸びたチャンネルTOP3」のような複数の切り口を用意し、1人あたりの閲覧ページ数を増やす(ソーシャルXPが1人あたり6〜10ページ見られてたのはこの設計のおかげ)
  - チャンネル本人(YouTuber)がリツイート/引用してくれると一気に伸びる。動画説明欄に載せてもらえるよう、目立つチャンネルには「あなたのページはこちらです」的なDMを送ってみるのもアリ(営業苦手なら定型文を1つ作っておいて使い回す)

### ③ 週刊まとめ記事の自動生成(SEOロングテール流入)
- 「今週の新台バズランキング」「今週伸びたパチスロ系YouTuberTOP5」を、収集済みデータから**自動でテキスト生成**して記事化するページを追加
- 過去のパチンコ新台Bot構想とも合流できる。攻略情報ゼロ・事実データの並び替えだけなので方針に抵触しない
- Google検索での「〇〇(YouTuber名) 登録者数」のようなロングテールキーワードを拾える

### ④ ノウハウの商品化(別ルートの収益、副産物)
- 「YouTuber系トラッカーサイトの作り方(データ収集の自動化設計)」を note やBrainで数千円で販売する、という横展開
- 今回作った設計(YouTube検索APIを使わずquotaを節約する構成など)はそのまま再利用可能な知見。AdSense収益が育つ前の"つなぎ"としても機能する
- ただし本業(パチログ運用)を圧迫しない範囲で、余力があれば

### ⑤ ホール公式・メーカーとの提携広告(将来、実績ができてから)
- アクセス数が一定水準に育ったら、当初構想にあった「ホール公式広告」枠を検討
- 現段階では優先度低(信頼と実績が先)

## 3. 今のプロダクトに足すと効果的な機能(優先順位)

1. **週刊まとめページ(③)** — 収集データをそのまま流用でき、実装コストが低い割にSEO効果が高い
2. **Xシェア時の見栄えの改善** — 今の画像生成をもう少しリッチに(グラフの推移も画像に入れるなど)。シェアされた時のクリック率に直結
3. **メール/LINE通知機能** — 「推しチャンネルが急上昇したら通知」のような軽いプッシュ機能。実装コストは高めなので、AdSense収益が伸び始めてから検討

## 4. タイムライン目安(1日1〜2時間ペース)

| 期間 | やること |
|---|---|
| 1〜2週目 | n8nでの定期収集を自動化。チャンネル数を20→50件程度に拡充 |
| 3〜4週目 | 週刊まとめページを実装。Xでの手動シェア・投稿を習慣化(週1〜2回) |
| 1〜2ヶ月目 | 独自ドメイン取得・AdSense申請。データが2〜3週間分溜まって推移グラフが機能し始める |
| 2〜3ヶ月目 | アクセス数を見ながら新台マスターデータ・チャンネル数をさらに拡充。AdSense審査通過後、広告配置の最適化 |
| 3ヶ月目〜 | 月間PV2万前後を目標に、Xシェア文言・回遊導線をABテスト的に改善 |

## 5. 忘れてはいけない制約

- 攻略・期待値・立ち回り・設定判別は今後も一切扱わない(サイトの安全性・信頼性の根幹)
- 課金・契約を伴う施策(Stripe、独自ドメイン以外の有料ツール等)は、収益が出る前は極力避ける
- 営業・動画編集・顔出しが必要な施策(YouTube進出、インタビュー企画等)は基本的に採用しない。テキスト・データ・自動化で完結する施策を優先する
